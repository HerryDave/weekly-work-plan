
## 1. 项目定位
面向**室经理**与**组长**，以**周**为周期管理多项目环境下的人力投入（人天），实现计划、跟踪、预警一体化。

## 2. 用户角色与权限

| 角色     | 权限说明                                                                 |
| -------- | ------------------------------------------------------------------------ |
| **室经理** | 查看全室数据；管理所有项目；直接编辑任何组的周计划（操作留日志）；审批跨组支持申请（若启用审批流）；接收重大预警。 |
| **组长**   | 管理本组人员；制定并编辑本组员工的周计划；**统一录入本组员工的实际投入**；可发起跨组支持申请；处理本组相关预警。 |
| **员工**   | **只读**查看自己的周计划与实际投入历史（无录入/修改权限）。                 |

> **特别确认**：
> - 实际投入由**组长统一录入**，员工不参与填报。
> - 跨组支持采用**简化方案**：项目负责人（可以是员工）直接添加跨组成员，系统自动通知对方组长，无需审批。

## 3. 核心功能模块

### 3.1 组织架构管理
- 一个室下设多个组（如技术室一组、技术室二组）。
- 每个用户属于一个组（室经理可不属于具体组）。
- 室经理可增删改组；组长可管理本组人员（添加/移除成员）。

### 3.2 项目管理
- **项目属性**：名称、负责人（可为员工）、类型（组内/跨组）、状态（筹备中/进行中/已结项）、起止日期。
- **周人力需求（可选）**：可为项目某周设置期望人天，用于预警。
- **项目成员**：
  - 组内项目：自动包含本组所有成员。
  - 跨组项目：项目负责人可直接添加任意组的员工为成员，**系统自动发送站内通知给该员工所属组长**（无需审批）。
- 支持项目结项归档。

### 3.3 周计划管理
- **时间单位**：以周为粒度，周一至周日。
- **计划数据**：某员工在某周投入某项目的人天数（支持0.5人天）。
- **操作权限**：
  - 组长：为本组任何员工制定/修改周计划。
  - 室经理：可修改所有组的计划（操作留日志）。
  - 员工：只读。
- **主要视图**：
  - **组计划看板**：表格形式（员工×项目×周），支持批量编辑。
  - **项目人力视图**：显示某项目在各组、各周的计划投入总和。
  - **个人视图**：员工查看自己的每周计划分布。

### 3.4 实际人力投入录入
- **录入人**：组长（或室经理代录）。
- **录入方式**：组长按周选择员工，填写该员工在各项目上的实际人天数。
- **对比分析**：自动计算计划 vs 实际偏差（绝对值与百分比）。

### 3.5 预警管理（系统内提醒）
- **预警规则**（MVP实现前两条）：

| 编号 | 名称               | 触发条件                                                       | 推送对象               |
|------|--------------------|---------------------------------------------------------------|------------------------|
| W01  | 项目周人力不足     | 某项目某周计划总人天 < 该项目该周需求人天（若需求已设置）         | 项目负责人、室经理     |
| W02  | 个人过度负载       | 某员工某周计划总人天 > 阈值（默认5人天，可配置）                | 该员工所在组长         |
| W03  | 跨组支持通知       | 项目负责人添加跨组成员（简化方案无审批，仅通知）                | 被添加成员的组长       |
| W04  | 连续偏差过大       | 某员工连续两周计划vs实际偏差绝对值 > 30%                        | 组长、室经理           |

- **预警呈现**：系统内红点/列表，支持标记已读/处理。
- **操作**：点击预警可跳转至相关页面（如项目周需求编辑页）。

### 3.6 跨组支持流程（简化确认版）
1. **项目负责人**（可为员工）在项目成员管理页面，直接添加任意组的某员工。
2. 系统自动创建**站内通知**，发送给该员工所属的组长，内容：“您组内员工[姓名]已被添加到项目[项目名称]”。
3. 组长无需审批，可在周计划中正常为该员工分配该项目投入。
4. 若需移除成员，项目负责人同样直接删除，系统再次通知组长。

> 此方案省略审批环节，提高效率，室经理仍可从全局视图监控所有项目成员。

### 3.7 报表与统计（辅助）
- **室人力总览**：各组每周计划总人天 vs 实际总人天。
- **项目人力跟踪**：项目每周计划与实际累计对比。
- **个人负载排行**：按周列出计划人天最高的员工（辅助发现过载）。

## 4. 业务流程示例（含跨组支持）

> 场景：跨组项目“智能系统升级”需要技术室一组员工张三支持，W5-W7每周2人天。

1. 项目负责人（例如技术室二组组长）在平台添加张三为项目成员。
2. 系统自动通知一组组长：“您组内员工张三已被添加到该项目”。
3. 一组组长在制定W5周计划时，为张三分配该项目2人天，同时可减少其他任务。
4. W5结束后，一组组长录入张三在该项目的实际投入（如1.5人天）。
5. 系统自动对比计划与实际，若偏差较大触发W04预警（若启用）。
6. 室经理可查看所有人力投入情况，必要时干预。

## 5. 非功能性需求摘要
- **易用性**：周计划支持表格内联编辑，批量保存。
- **响应速度**：周计划页面加载 ≤ 3秒（40人×20项目）。
- **数据安全**：基于角色的数据隔离，后端强制校验权限。
- **可追溯**：重要操作（计划修改、实际录入、成员变动）记录日志。

| 模块               | 包含内容                                                                 | 备注                     |
| ------------------ | ------------------------------------------------------------------------ | ------------------------ |
| 组织与用户         | 组、用户（角色：经理/组长/员工）                                          | 基础数据                 |
| 项目管理           | 创建/编辑/查看，支持跨组项目成员手动添加（简化流程，无审批）               | 成员变动自动通知组长     |
| 周计划             | 组长为本组员工制定计划；室经理只读+可编辑；表格批量编辑                    | 支持计划人天录入         |
| 实际投入           | 组长按周录入本组员工实际人天；自动对比计划                                | 员工不参与录入           |
| 预警               | W01（项目周人力不足）、W02（个人过度负载）                                | 系统内提醒               |
| 报表               | 人力总览表（各组计划/实际总和）                                           | 简单统计                 |
